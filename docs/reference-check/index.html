Reference-Check Demo

Check your citations against your reference list, then check whether each source actually supports the claim it backs.

  1. Motivation
  2. Set up
  3. Reference check
  4. Source-claim check
  5. Check by hand

Motivation

Two things go wrong with citations. A citation can point to a source that does not exist. Or it can point to a real source that does not say what you claim.

This demo catches both, on a tiny manuscript with planted errors. The two checks need different things. The reference check works against your reference list alone. The source-claim check also needs the sources themselves, kept as a small local knowledge base.

One rule. Every flag is something to go check yourself, not a verdict to publish.

Set up

Install Claude Code and the skills first (see Getting started). Then clone the demo:

git clone https://github.com/scdenney/ai-for-research.git
cd ai-for-research/demos/reference-check

Reference check reference list only

Does each citation exist, resolve, and match an entry? This needs only the manuscript and the reference list, no knowledge base.

/oss:citation-check sample-project/manuscript.md sample-project/references.bib

On this project it flags, among others:

WhatVerdict
othen2022 — a real-looking article that was never writtenLikely fabricated
vance2018 — cited in the text, no entry in the listParity
kowalski2017 — in the list, never citedParity
hainmueller2014 — a real methods paperVerified

The last row is the control: one real reference that checks out, so you can see the difference between a flag and a clean pass.

Source-claim check needs a knowledge base

Does the cited source actually support the claim? This check reads the sources, so it needs them on disk as a small knowledge base in sources/md/ (already included here). It refuses to run if that is missing, rather than guess.

/oss:fact-check sample-project/manuscript.md

On this project it flags:

ClaimVerdict
"compulsory voting also raises political knowledge"Contradicted
"higher social trust causes higher turnout"Overclaim
"a modest, short-lived rise in youth turnout"Supported

A citation can be real, well-formed, and resolve cleanly, and still be attached to a claim its source never makes. Only reading the source catches that.

Check by hand

The reports are where to start looking, not the last word. Open the source a report quotes and read it. A "likely fabricated" label can be a real paper the tool could not reach. A "causes" that should read "is associated with" is a one-word fix only you can make.

Every planted error is listed in ANSWER-KEY.md. Then try breaking it: add a real reference and watch it verify, fix a claim and watch Contradicted turn to Supported, or delete a source file and watch the source-claim check refuse to run.
